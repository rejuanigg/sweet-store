# Sweet store
It's a full-stack ecommerce platform built with Laravel and Vue.js. 

## features
- ** Role-based access control ** -- Admin roles (employed, owner), Customer role (client). 

- ** Products Management ** -- If the user has an admin role, they can create, update, delete products, they can also create, update, and delete product sections and categories.

- ** Orders Management ** -- If the user has an admin role, they can update the order status, the statuses are "processing", "waiting", "completed" and "canceled". The users can track status changes in their orders view. 

- ** Ratings ** -- The users can rate products using a five-star system, and all the users can view the average rating for each product.

- ** Filter Products ** -- The products are filtered by section and category. Each section contains multiple categories, allowing users to browse products by their preferred section.

## tech stack
- ** Backend ** Php, Laravel.
- ** Frontend ** Vue.js, Tailwind
- ** Auth ** Laravel sanctum
- ** Deploy ** Render (backend), Vercel(frontend)

## why use laravel?
I used laravel because it's a big framework with many tools to create a web app.
Personally, laravel has a low learning curve.
Also, laravel documentation is very easy to learn, and provides practical, easy-to-follow examples.

## why use SQL?
I chose MySQL because it's a very strong relational database, where I can understand the relations, the entities, the values, in a straightforward way.
